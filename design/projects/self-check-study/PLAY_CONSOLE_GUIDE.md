## [1] 새 앱 만들기

Console 우상단 **앱 만들기** 버튼 → 다음 화면:

```
┌──────────────────────────────────────────────────────────┐
│ 앱 세부정보 입력                                          │
├──────────────────────────────────────────────────────────┤
│ 앱 이름 *           : 셀프체크 스터디            (50자)   │
│ 기본 언어 *         : 한국어 - ko-KR                      │
│ 앱 또는 게임 *      : ⦿ 앱                                │
│ 무료 또는 유료 *    : ⦿ 무료   (※ 무료→유료 변경 불가)   │
│ 선언 *              : ☑ Developer Program Policies 준수   │
│                       ☑ 미국 수출법 준수                  │
│                                                           │
│                              [ 취소 ]  [ 앱 만들기 ]      │
└──────────────────────────────────────────────────────────┘
```

→ "앱 만들기" 누르면 좌측 메뉴 트리 + 대시보드로 이동.

## [2] 앱 콘텐츠 (좌측 메뉴 → "앱 콘텐츠")

각 항목을 순서대로 작성. 완료하면 항목 옆에 초록 체크가 뜬다.

| 항목 | 입력값 |
|---|---|
| 개인정보처리방침 | URL: `https://woo-hoo.kr/privacy` |
| 광고 | "아니요, 앱에 광고가 없습니다" |
| 앱 액세스 권한 | "모든 기능을 제한 없이 사용할 수 있습니다" (로그인 없음) |
| 콘텐츠 등급 | 설문 작성 → 결과: **전체 이용가** (이메일 입력 후 카테고리=교육/참고) |
| 타겟층 및 콘텐츠 | 타겟 연령대 **13세 이상** 선택 |
| 데이터 보안 | 아래 표 그대로 응답 |
| 정부 앱 | "아니요" |
| 금융 기능 | "내 앱에서 금융 기능을 제공하지 않음" |
| 건강 앱 | 해당 없음 |

### 데이터 보안 양식 (copy.md 동일)

| 질문 | 응답 |
|---|---|
| 데이터를 수집하거나 공유합니까? | **아니요** |
| 사용자 데이터가 전송 중 암호화됩니까? | 해당 없음 (외부 전송 없음) |
| 사용자가 데이터 삭제를 요청할 수 있습니까? | 예 (앱 삭제 시 즉시 파기) |

→ "데이터를 수집하거나 공유합니까? = 아니요"를 고르면 나머지 세부 항목은 대부분 생략된다.

## [4] 프로덕션 트랙 → 새 버전 만들기

좌측 메뉴 → **프로덕션** → "새 버전 만들기"

1. **App Bundle 업로드**: `build/app/outputs/bundle/release/app-release.aab`
   - 첫 업로드 시 **Play 앱 서명** 사용에 동의 (Google이 서명 키 관리, 업로드 키는 내 keystore).
2. **출시명**: 자동 채워짐 (`9 (0.4.1)`) — 그대로 둬도 됨.
3. **출시 노트** (`<ko-KR>` 안에 입력):

```
첫 출시예요!
- 요일별 학습 일정 + 날짜별 인라인 달력 편집
- 드래그앤드롭으로 세션 순서 자유 변경
- 타이머 + 만족도/도달률 기반 점수
- 도장 그리드 · 무지개 달력 · 누적 통계
- 시작/자유시간 알림 (오프라인 로컬)
- 카톡 공유 3 진입점
```

4. "다음" → "저장" (아직 출시 아님).

> 국가/지역: 별도 "국가/지역" 설정에서 **대한민국** 선택 (또는 전체 국가). 첫 등록이면 한국 단일로 시작 권장.

## [5] 검토를 위해 전송

좌측 메뉴 → **대시보드** 또는 **출시 개요**에서 미완료 항목 확인.

- "앱 설정" 섹션의 모든 항목에 초록 체크가 떠야 함.
- 프로덕션 트랙의 "검토를 위해 전송" 버튼 활성화 → 클릭.
- 심사 기간: 첫 앱은 평균 **1~7일** (Google 정책 검토 포함, 보통 1~3일).

## 자주 막히는 지점

| 증상 | 원인 / 해결 |
|---|---|
| AAB 업로드 시 "서명되지 않음" 오류 | release 빌드가 key.properties로 서명됐는지 확인. debug AAB 업로드 금지. |
| "패키지 이름이 이미 사용 중" | `kr.woohoo.self_check_study`는 신규. 다른 계정/이전 등록과 충돌 시 패키지명 변경 필요(피해야 함). |
| 콘텐츠 등급 설문 후에도 빨간 X | 설문 제출 버튼까지 눌러야 등급이 확정됨. |
| 검토 전송 버튼 비활성 | "앱 콘텐츠" 항목 중 하나라도 미완료. 대시보드에서 빨간 항목 추적. |
| 데이터 보안 거부 | "수집 안 함"이어도 양식 자체는 **제출**해야 체크됨. |
| "API 수준 35 이상 타겟팅 필요" | targetSdk 34 → 35 로 빌드 필요. compileSdk 35 는 AGP 8.1+ 요구 → Gradle/AGP/Kotlin 동반 업그레이드(이미 v0.4.1+8 에서 적용됨). |
| "N 버전 코드는 이미 사용되었습니다" | 한 번 업로드된 versionCode 는 **영구 재사용 불가**(초안/취소 포함). `pubspec.yaml` 의 `+N` 을 더 큰 수로 올려 재빌드. |
| privacy URL "SSL 인증서 없음" | GitHub Pages custom domain 인증서 미발급. repo Settings>Pages 에서 도메인 재등록 후 "Certificate provisioned" 대기(최대 24h). |
