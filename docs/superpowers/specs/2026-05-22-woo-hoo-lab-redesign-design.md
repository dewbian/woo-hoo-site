# woo-hoo.kr 리디자인 설계 — "프로덕트 쇼케이스 + 1인 실험실"

- **작성일:** 2026-05-22
- **대상:** `woo-hoo.kr` (GitHub Pages 정적 호스팅, vanilla HTML/CSS)
- **목표:** 단순 개인정보처리방침 호스팅용 페이지를, woo-hoo 브랜드(1인 실험실)의 프로덕트 쇼케이스 + 빌딩 인 퍼블릭 사이트로 전환

---

## 1. 컨셉

woo-hoo는 한 사람이 운영하는 작은 앱/실험 모음 도메인이다. 사이트는 두 가지를 동시에 한다:

1. **쇼케이스** — 현재 밀고 있는 대표 프로덕트(셀프체크 스터디)를 전면에.
2. **실험실(building in public)** — 다른 앱·실험, 지금 만드는 중인 것, 개발 로그를 공개.

브랜드명/표기: **`woo-hoo.lab`** (로고), 도메인은 `woo-hoo.kr`.

## 2. 비주얼 방향 — "Loud 하이브리드"

에디토리얼/랩 노트 구조 + 두꺼운 아웃라인 타이포 + 손그림 액센트. 얌전하지 않고 개성 있게.

- **베이스:** 페이퍼 톤 `#f4f1ea`, 잉크 `#111`
- **포인트 컬러:** 블루 `#0a84ff`, 핑크 `#ff4da6`, 옐로 `#ffd60a`, 그린 `#9cff6b`, 스카이 `#7cd6ff`
- **타이포:** 헤비 디스플레이(Pretendard Black/ExtraBold, CDN). 큰 헤드라인 + `letter-spacing` 타이트 + 일부 텍스트 아웃라인(`-webkit-text-stroke`).
- **테두리/모양:** `2px solid #111` 보더, 라운드 코너(8~14px). 카드/버튼에 일관 적용.
- **손맛 액센트:** 인라인 SVG 낙서(부엉이, 별), 회전된 "woo!hoo!" 텍스트. 과하지 않게 포인트로만.
- **히어로만 비비드 단색 필드(블루)**, 나머지는 페이퍼 톤으로 차분히 — 대비로 대표 프로덕트를 강조.

## 3. 페이지 구조

### 3.1 `index.html` — 싱글 페이지 (스크롤)

스크롤 순서:

1. **Nav (sticky)** — `woo-hoo.lab` 로고 / 앵커: 실험실(#products)·개발로그(devlog.html)·다운로드(#download)·구독(#subscribe). 모바일은 햄버거 토글.
2. **Hero — 대표 프로덕트 (셀프체크 스터디)**
   - 블루 배경, "★ NOW FEATURING — 심사 중" 배지
   - 큰 타이포 헤드라인(셀프체크 스터디), 한 줄 설명
   - CTA: `출시 알림 받기 ↓`(#subscribe로 스크롤), `자세히`(#products)
   - 폰 목업 + woo!hoo! 낙서 액센트
3. **실험실 그리드 (#products)** — 카드 3개, 상태 배지 포함
   - 셀프체크 스터디 — `● 심사 중` (블루) — 학습 스케줄 체크 앱
   - make-money — `● 운영 중` (그린) — 뉴스 자동 크롤링→관점별 콘텐츠 생성→광고 삽입→블로그/티스토리 자동 등록
   - googsky — `● 만드는 중` (옐로 카드) — 대입 컨설팅 앱
   - 각 카드: 이름 / 한 줄 설명 / 상태 배지 / 링크(placeholder)
4. **지금 만드는 중 (Building now)** — 다크(`#111`) 배경, googsky 강조
   - 컨셉 설명: "과목별 등급만 가볍게 입력하면 안정·적정·도전권 대학을 나눠서 추천"
   - 진행도 바(시각용, 정적 값) + "개발로그에서 따라가기" 링크
5. **개발 로그 미리보기** — 최근 2개 카드(날짜+제목) + "전체 보기 → devlog.html"
6. **다운로드 & 구독 (#download / #subscribe)** — 스카이 배경
   - 스토어 배지: App Store / Google Play (셀프체크 스터디. 심사 중이라 비활성/"준비 중" 상태 표기 가능)
   - 이메일 구독 폼 (Formspree 연동 — endpoint placeholder)
7. **푸터** — © 2026 woo-hoo.kr · 개인정보처리방침(privacy.html) · GitHub · X · Email

### 3.2 `devlog.html` — 별도 개발 로그 페이지

- 동일 비주얼 시스템(페이퍼 톤, 보더, 타이포)
- 상단 nav: `← woo-hoo.lab`
- 타임라인 형태: 날짜 + 제목 + (선택)본문. 항목은 HTML에 직접 작성(정적).
- 프로젝트 태그(셀프체크/ make-money / googsky)로 구분.

### 3.3 `privacy.html` — 기존 유지 + 스타일 정렬

- 내용 그대로, 비주얼 시스템(헤더/푸터/폰트/색)만 새 디자인에 맞게 정렬해 일관성 확보.

## 4. 기술 / 제약

- **빌드 없음.** vanilla HTML + CSS(+ 최소 JS). GitHub Pages `main` 루트 정적 호스팅 유지.
- **폰트:** Pretendard CDN(헤비 weight). 실패 시 시스템 폰트로 폴백.
- **JS(최소):** 모바일 nav 토글, 부드러운 앵커 스크롤. 폼은 기본 HTML form submit(Formspree).
- **CSS 전략:** 공통 디자인 토큰을 한 곳에 정의. 페이지가 3개뿐이므로 외부 `styles.css` 1개로 분리해 공유(현재 인라인 → 공유 스타일시트로 전환). 색/보더/타이포를 CSS 변수로.
- **이미지/아이콘:** 외부 의존 최소화. 낙서·아이콘은 인라인 SVG. 폰 목업은 CSS 또는 SVG로(이미지 파일 의존 회피).
- **반응형:** 모바일 우선. 그리드는 데스크탑 3열 → 모바일 1열.
- **접근성:** 색 대비 확인(특히 블루/화이트, 옐로/잉크), `alt`/`aria-label`, 키보드 포커스.

## 5. Placeholder 처리 (사용자가 실제 값 추후 제공)

아래는 자리만 잡고, 눈에 띄게 표시(주석 + 명확한 더미 값)해 교체가 쉽도록 한다:

- 셀프체크 스터디 App Store / Google Play 링크
- make-money 링크(웹/레포)
- googsky 링크(레포/랜딩)
- GitHub, X(트위터) 프로필 URL
- Formspree 폼 endpoint (`https://formspree.io/f/XXXXXXXX`)
- 이메일: `dev@b-lot.co.kr` (확정값 — 사용)

각 placeholder에 `<!-- TODO: 실제 링크로 교체 -->` 주석을 단다.

## 6. 콘텐츠 사실(확정)

- 도메인/브랜드: woo-hoo (토탈 관리 도메인 = 실험실 브랜드)
- 셀프체크 스터디: 학습 스케줄 등록·달성 체크 앱, **현재 스토어 심사 중**. (privacy.html이 이 앱의 처리방침)
- make-money: 뉴스 자동 크롤링 → 관점별 콘텐츠 생산 → 광고 삽입 → 블로그/티스토리 자동 등록 자동화
- googsky: **현재 작업 중**. 과목별 등급 입력 → 안정권/적정권/도전권 대학 분류 추천(대입 컨설팅)

## 7. 범위 밖 (YAGNI)

- 백엔드/서버, DB, 로그인 — 없음(정적)
- CMS/마크다운 빌드 파이프라인 — 없음(devlog는 HTML 직접 작성)
- 다국어 — 한국어 only
- 애니메이션 라이브러리 — 없음(CSS transition 정도)

## 8. 성공 기준

- 첫 화면에서 "이건 셀프체크 스터디를 미는 1인 실험실"이 즉시 읽힘
- 한 페이지 스크롤로 프로덕트·실험·진행중·로그·구독이 모두 노출
- 모바일/데스크탑에서 깨짐 없음, 레퍼런스(Honk/al-murphy) 수준의 개성·완성도
- placeholder만 실제 값으로 바꾸면 바로 배포 가능한 상태
